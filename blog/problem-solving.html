A space too far

I wanted to format my wellness plan content into a table, and thought this would be a good opportunity to reinforce what I had learned from Sprint 2 by implement padding within the cells and margins within tables. However, I initially couldn't get the table padding and margins to work properly. The console wasn't showing any errors when I saved my code, but the CSS wasn't working as I had expected. The table lacked any padding or margins at all, regardless of how extreme I set the CSS padding and margin values to.

I closely compared my code to examples provided in HTML documentation but couldn't see any difference between my own code and theirs, so I knew the odd behaviour of the CSS was likely due to a silly mistake. I continued to look line by line at my CSS code and finally discovered that I had included a white space between the value and the units of the attribute (e.g. "10 em" vs "10em"). This was after a fairly significant amount of time spent trouble shooting, but I didn't feel much frustration as I was expecting the error to be due to something like this in the first place. I also learned that VS Code colour-codes units and values in CSS, which is a simple way to see if there are any errors similar to my own at a glance.

Nested functions

One of my solutions that I thought was particularly elegant was my solution to the 'Find You' problem in JS Kata 5. The problem wanted me to define a function that used the find() method to locate an element within a group of elements. The find() method itself involves calling a separate function on each element within the group, and then returning the element for which the separate function returns 'true'. I therefore realised that I would need to find some suitable function to provide to the find() method, or I would need to define a function of my own. This was more appealing to me, but this meant that I had the problem of how - or more specifically where - to define the separate function used in the find() method.

To solve this problem one way, I could define the separate function globally and then provide it to find() in my main function, but this would mean that mmy main function wouldn't be self-contained or modular. Based on my understanding, I wouldn't be able to export my primary function without also exporting the global secondary function function. I then remembered that we had previously learned that functions could be defined within other functions. I therefore defined the second function used with the find() method withinthe primary function. I thought this was an elegant solution because the end result (i.e. the primary function) was entirely self-contained. It felt satisfying to solve this and reinforced what we had previously learned about different ways to define functions in Javascript.
